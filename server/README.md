# Voxel Wars — авторитетный сервер (Colyseus)

Миграция с «браузер-хост» на **server-authoritative** на базе [Colyseus](https://github.com/colyseus/colyseus).
Симуляция крутится на сервере (Node/V8), клиент только рисует стейт и шлёт команды.
Стейт синхронизируется бинарными дельтами Colyseus; команды валидируются на сервере (анти-чит).

## Запуск

```bash
npm install
npm start                 # WS + HTTP-API на :2567 (один порт)
```

Переменные окружения:

| env | по умолчанию | назначение |
|---|---|---|
| `PORT` | 2567 | порт WS + HTTP |
| `REDIS_URL` | — (Local) | напр. `redis://host:6379` → масштаб на N процессов |
| `JWT_SECRET` | dev-secret | подпись токенов (обязательно сменить в проде) |
| `DB_FILE` | `data/db.json` | файловый стор аккаунтов (в проде → Postgres) |

HTTP-API: `POST /auth/register`, `POST /auth/login` → `{token, user}`; `GET /leaderboard`; `GET /health`.
В игру входят с токеном: `client.joinOrCreate('game', { token })` (без токена — гость).

## Богатый клиент: game.html на Colyseus

Оригинальный воксельный `game.html` подключается к этому серверу как тонкий клиент
(переиспользует guest-рендер: города/регионы/призраки-юниты/HUD кормятся стейтом Colyseus,
действия транслируются в команды). Запуск:

```
http://localhost:3000/game.html?mp=ROOM&cs=1     # mp= активирует guest-режим, cs=1 — мост на Colyseus
```

Мост в конце `game.html`: захватывает внутренний guest-`onMsg`, кормит его снапшотами из
Colyseus-стейта (`applySnap`/`reconcile`), а `MP.cmd` → `room.send(...)`. Выбор страны →
join за эту фракцию (сервер чтит `options.faction`). Локальный сим заморожен — всё авторитетно.
(Минимальный клиент-кубики `mp-client.html` оставлен для отладки.)

## Тесты

```bash
npm test                  # всё (sim + e2e + meta), exit-код для CI — 119 тестов
```

`test/sim.test.js` (75) — экономика, производство, манпауэр, покупка/прокачка, осада→

флот/авиация (постройка/спавн/движение/грид-бой), анти-чит.
`test/e2e.test.js` (14) — назначение фракций, синк стейта, команды по сети, анти-чит,
осада, движение отрядов по карте, постройка флота+авиации, кросс-клиентский синк, изоляция.
`test/meta.test.js` (14) — регистрация/вход (JWT), авторизованный вход в игру, гость,
матчмейкинг (листинг/метаданные/joinById/лобби), реконнект, персист W/L/рейтинга, лидерборд.

Версии запиннены на стабильную линию **0.16** (сервер `colyseus@0.16.5` + клиент `colyseus.js@0.16.22`);
`latest`-теги в npm разъехались (сервер 0.17 без парного клиента), поэтому пин обязателен.

## Структура

```
index.js        — бутстрап: WS + HTTP-API на одном порту, presence/driver (Redis|Local), лобби
schema.js       — Colyseus-схема (City/Squad/Ship/Plane + дипломатия) — то, что синкается клиенту
GameRoom.js     — комната: Sim @15Гц, проекция sim→схема, команды, onAuth, реконнект, итоги матча
auth.js         — scrypt-пароли + HS256-JWT (без внешних зависимостей)
db.js           — персист аккаунтов/матчей (файловый, → Postgres)
http-api.js     — HTTP: register / login / leaderboard / health
sim/            — ЧИСТАЯ симуляция (без Three/DOM/Colyseus), переносимо и тестируемо
  constants.js · City · Squad · Ship · Plane · SpatialGrid
  tech.js + tech-data.js   — древо технологий (58 узлов)
  water.js + water-data.json — водный грид (для флота)
  map-data.json            — карта Европы (143 города, 240 рёбер)
  Sim.js                   — мир: тик, экономика, дипломатия, технологии, движение, флот/авиация
test/           — sim.test.js (75) · e2e.test.js (14) · meta.test.js (14) · harness.js
bench-combat.js — бенч O(n²) vs spatial-grid
```

## План миграции

- [x] **Фаза 0** — каркас Colyseus: авторитетный тик, синк стейта, канал команд, идентичность, анти-чит.
- [x] **Фаза 1a** — стратегический слой: экономика, производство, манпауэр, осада/захват/оккупация.
- [x] **Фаза 3 (срез)** — браузер-клиент `../tiny-world-builder/mp-client.html`: Three.js рисует авторитетный стейт + команды. Тик 15 Гц.
- [x] **Фаза 2 (бой/сетка)** — `sim/SpatialGrid.js` + `bench-combat.js`: O(n²)→O(n). Вшито в живой морской/воздушный бой.
- [x] **Фаза 1b** — ПОЛНОСТЬЮ:
  - Дипломатия: война/мир/союз/поддержка/разрыв, политочки, мобилизация, перемирие, репарации, оккупация→аннексия.
  - Древо технологий: 58 узлов (`sim/tech-data.js`), исследования, слоты, эффекты на города/юниты/манпауэр.
  - Реальная карта Европы (`sim/map-data.json`): 143 города, 24 страны, граф 240 рёбер (road/ferry/pass).
  - Движение отрядов по графу (Дейкстра + `Squad`), полевой бой, прибытие→подкрепление/осада.
  - Флот (`Ship`, водный грид `sim/water-data.json`) + авиация (`Plane`): постройка на верфи/аэродроме, движение, грид-бой.
  - Клиент рисует отряды/корабли/самолёты; дипломатия и политочки в схеме.
- [x] **Фаза 4** — мета и масштаб:
  - Аккаунты: `auth.js` (scrypt + HS256-JWT), `db.js` (персист, → Postgres), `http-api.js` (register/login/leaderboard).
  - Авторизация входа: `GameRoom.onAuth` (JWT → identity, без токена → гость).
  - Матчмейкинг: метаданные комнат (`setMetadata`), realtime-листинг, `LobbyRoom`, joinById.
  - Реконнект: `allowReconnection` (30с — фракция/сессия сохраняются при обрыве).
  - Итоги матча: запись W/L и рейтинга при выбывании/победе (очередь `sim.eliminations`).
  - Масштаб: Redis presence/driver через `REDIS_URL` (несколько процессов за балансировщиком), локальный fallback.
- [x] **ИИ-оппоненты** — порт `aiActFaction`: незанятые фракции воюют, заключают союзы/мир, исследуют, набирают и двигают армии, захватывают города (`Sim.aiUpdate`, opt-in `ai:true`).
- [x] **Постройка верфи/аэродрома** в любом городе (`cmdBuildYard`): верфь — в прибрежном, аэродром — в любом; даёт умение строить корабли/самолёты. Флот/авиация доступны всем странам.
- [x] **Бой-полиш**: обстрел берега кораблями (`shipBombard`, tech shipMissile), бомбёжка городов авиацией (`planeBomb` + приказ `cmdAirOrder`, tech planeBomb), башни atk-городов (`cityTowers`), ПВО (`cityAA` + `cmdBuildAA`). ИИ строит ПВО.

Полная игра перенесена на авторитетный сервер. Дальше — продакшен-инфраструктура (реальный Redis-деплой, БД, CI/CD) и контент.

## Масштабирование на несколько процессов

```bash
# каждый процесс — свой порт, общий Redis для presence/room-discovery
REDIS_URL=redis://localhost:6379 PORT=2567 node index.js
REDIS_URL=redis://localhost:6379 PORT=2568 node index.js
# ... за балансировщиком (sticky по WS). Колизей сам находит/распределяет комнаты через Redis.
```

Один матч (24 игрока, ~1680 юнитов) = единицы мс/тик (см. бенч) → десятки матчей на ядро.
Горизонтально процессы независимы; состояние матча живёт в одной комнате (одном процессе),
Redis отвечает за discovery/seat-reservation между процессами.

## Замер боя — Фаза 2 закрыта (`node bench-combat.js`)

Воздушный бой, dt=0.066 (15 Гц), 24 владельца все против всех:

```
— равномерно по карте (реалистично) —
n= 480  naive 0.61мс → grid 0.05мс  (12.9×)  foe-mismatch=0
n=1200  naive 3.82мс → grid 0.08мс  (45.5×)  foe-mismatch=0
— «свалка» 40×40, без движения (вырожденный) —
n=1200  naive 4.05мс → grid 0.02мс  (209×)
```

Сетка даёт **идентичный таргетинг** (mismatch=0) и держит O(n) даже в куче.
Замеченное ранее в браузере «air 31.6мс» = бой + `plane.update()` с записью в Three-меши;
на чистом сервере (без Three) даже naive падает до ~3.8мс, а с сеткой — до 0.08мс.
Вывод: полный масштаб (~1680 юнитов) на сервере ≈ единицы мс/тик → десятки матчей на ядро в полном Node.
