## Что это
Прототип игры — гибрид **Mushroom Wars** (RTS-захват: города производят голду, покупаешь армию,
шлёшь рой юнитов захватывать соседние города) **+ grand strategy** (карта Европы, 24 страны-фракции,
дипломатия). 3D-воксельный стиль на движке Tiny World Builder (Three.js r128).

Похожие игры: Rise of Nations (Roblox), Mushroom Wars 2, Supremacy 1914 / Call of War, Risk.

Папка проекта: `/Users/viacheslavmuliukin/mushroom-wars-clone/` (ВНЕ основного репозитория, как просил юзер).

(Старые версии `index.html`, `europe.html` — 2D-предки, не трогаем, но из europe.html взяты координаты городов/контуры стран.)

## КАРТА (география)
- **GRID=256, TILE=1** (мир 256×256 тайлов). Проекция lon/lat: `LON0=-13,LON1=51,LAT0=34,LAT1=70`.
- **24 фракции** (каждая страна = отдельный игрок/ИИ): Британия, **Франция (игрок)**, Испания, Португалия,
  Италия, Германия, Бенилюкс, Центр, Польша, Норвегия, Швеция, Финляндия, Дания, Балканы, **Греция**,
  Украина, Россия, Турция, Грузия, Армения, Азербайджан, Литва, Латвия, Эстония.
- **130+ городов** (реальные, из europe.html), `CITY_LIST` = `[name,lon,lat,size,_,country]`. Владелец = фракция страны.
- Полигоны стран в `COUNTRIES` (point-in-polygon → суша). Заплатки без городов закрывают «дыры»: **Альпы, Карпаты, Лапландия** (красятся по ближайшему городу).
- **Россия** соединена с Кавказом сухопутным «рукавом» (Северный Кавказ: Ростов, Краснодар, Сочи, Грозный, Махачкала, Астрахань).
- **Босфор**: Турция = 2 полигона (европейская Фракция со Стамбулом + азиатская Анатолия), между ними вода, связь паромом Стамбул↔Бурса.
- Острова: Пальма (Испания), Фареры (Дания), Сицилия (Палермо), Крит (Греция) — связаны паромами (`FERRY_LINKS`).
- **Цвета фракций** (`FACTION_COLOR`) подобраны алгоритмом «дальних точек», мин. перцептивная дистанция ≈100 (нет похожих).
- Рендер: вода = 1 плоскость с шейдером волн; суша = `InstancedMesh` (landTopIM/landPillarIM) по высоте (Perlin + горы Альпы/Карпаты/Кавказ/Скандинавия); + леса/скалы/облака (InstancedMesh). `assignRegions()` красит тайлы по владельцу (троттлинг через `markRegions()`/`regionsDirty`).

## МЕХАНИКИ
- **Экономика**: города дают голду владельцу (`gold[]` массив по фракциям). Армия покупается (`SOLDIER_PRICE=4`), очередь батчами.
- **Движение армий по графу дорог** (`EDGES/ADJ`, Дейкстра `findPath`): road / **pass** (горы ×0.5) / **ferry** (вода ×0.7). Через нейтралов прохода нет, через союзников — да. Армия не пролетает мимо чужого города — первый враждебный узел = цель.
- **Армия = рой кружочков** (`class Squad`, `unitsForCount` — число кружков ∝ размеру). `SQUAD_SPEED=4`.
- **Осады** (бой во времени, `City.siege`, орбы-армии у города), **полевые бои** (`fieldBattles` — отряды воюющих фракций дерутся на дороге).
- **Прокачка города за голду** (`upgradeCity`, 3 ветки prod/def/atk, тиры, `upgradeCost`). Меняет силуэт города (экономика=купол+монеты, оборона=стены+башни, атака=копья+знамёна).
- **Тех-дерево** (окно 🔬 / клавиша T): 5 веток (atk/def/eco/speed/prod ×4 тира), глобальные бонусы, `tech[]` по фракциям, `techMul`.
- **Дипломатия** (окно 🏛 Политика / P, + попап по клику на страну):
  - нейтралитет/война/союз (`relations`, `atWar/allied`). Атаковать можно ТОЛЬКО при войне.
  - **Мобилизация 60с** после объявления войны (`WAR_PREP`, `warReady/warCountdown`) — таймер в интерфейсе (слева список + в попапе).
  - **Переговоры о мире с данью** (`#peaceWin`): игрок предлагает дань, ИИ принимает/отклоняет (`peaceAcceptChance`).
  - **Предложение мира от ИИ игроку** (`#peaceOffer`, попап, ИИ откупается данью).
  - **Попап «вам объявлена война»** (`#warNotif`, очередь).
  - ИИ сам объявляет войны слабым соседям, просит мира если проигрывает.
- **Скорость игры**: пауза/1×/2×/3× (кнопки сверху, Space=пауза, F=скорость). `gameSpeed`, `gameTime`.
- **Win/lose**: остаться единственной живой фракцией / потерять все города.

## РОДА ВОЙСК (КЛЮЧЕВОЕ, последнее что делали)
1. **Армия** — по дорогам (см. выше).
2. **Флот** (`class Ship`, `ships[]`):
   - Производится на **Верфи** (`'Верфь Бордо'`, у Бордо, на побережье). Своя модель (`buildShipyard`): причал/кран/корпус. Связана дорогой с Бордо (захватываемая). Панель = только постройка кораблей (по 1, очередь).
   - Свободное движение по ВОДЕ (клик+тяни). `SHIP_SCALE=5`, `SHIP_SPEED=6`. Не заходит на сушу.
   - **Морской бой** (`navalBattles`): корабли воюющих фракций в радиусе стреляют (HP).
   - **★ Обстрел берега ракетами** (`SHIP_ATTACK_RANGE=16`): если в радиусе вражеский город/армия — корабль пускает **баллистические ракеты** (`class Missile`, дуга, `findShipTarget/fireMissile`). Урон городу (гарнизон, мин.1) или отряду (fcount). Эффекты взрыва/выстрела (`fx[]`, `spawnBlast/spawnMuzzle`, `updateMissiles`).
3. **Авиация** (`class Plane`, `planes[]`):
   - Производится на **Аэропорте** (`'Аэропорт Париж'`, у Парижа). Модель (`buildAirport`): ВПП/терминал/вышка. Связан дорогой с Парижем (захватываемый). Панель = постройка самолётов.
   - Свободный полёт НАД ВСЕМ (суша+вода). `PLANE_ALT=4.5` (чуть выше городов), `PLANE_SCALE=5`, `PLANE_SPEED=13`.
   - **Воздушный бой** (`airBattles`).

### Управление флотом/авиацией
- Унифицировано: `selectedUnits` (Set), `pickUnit`, `unitDrag`, `dragLead`.
- Клик по юниту = выбрать (Shift+клик = добавить). **Рамка** = массовый выбор юнитов в ней.
- Тянешь от выбранного юнита → **вся группа** движется в точку, расставляясь в сетку (`nearestWaterPoint` для кораблей).
- Выбранные юниты подсвечены голубым (emissive `tintMat`).
- **★ Линия движения юнита — жирная красная** (`unitDragTube` — цилиндр, т.к. тонкие WebGL-линии не толстеют; `showUnitDragLine`).

## ИИ
- `aiUpdate` → `aiActFaction(fid)` для каждой не-игроковой фракции: дипломатия (война/мир), тех, покупка армии, прокачка, атака реальной цели (первый чужой узел, только если мобилизация прошла).
- ⚠️ ИИ пока НЕ строит флот/авиацию (только игрок). Корабли/самолёты воюют между собой и обстреливают, если есть.

## МУЛЬТИПЛЕЕР (Colyseus server-authoritative)
- **Актуальный backend**: `server/index.js` / `server/GameRoom.js` на Colyseus. Старый `mp-server.js` relay удалён.
- **Запуск локально**:
  - backend: `cd server && npm start`
  - frontend: `cd tiny-world-builder && node tools/dev-server.js 3000`
  - игра: `http://localhost:3000/game.html?mp=ROOM`
  - лобби: `http://localhost:3000/` или `tiny-world-builder/index.html`
- **Модель**: сервер крутит `server/sim/Sim.js`; клиент не симулирует, а зеркалит Colyseus schema-state.
- **Bridge в `game.html`**: Colyseus state переводится в локальные `snap/ent` события для существующего Three.js рендера (`applySnap`/`reconcile`). В online-режиме `MP.guest=true`, браузер-хоста нет.
- **Назначение страны**: клиент выбирает страну, сервер отвечает `assigned{faction}`. Если страна занята, клиент переключается на фактически назначенную сервером фракцию.
- **Команды**: `buy/upg/send/war/ally/break/sup/peace/research/bship/bplane/shipmove/airorder/aa/yard`.
  Все они валидируются сервером через `Sim.cmd*`: владение, диапазоны id, ресурсы, война/союзы, `pct` отправки армии, условия мира.
- **Online мир с условиями**: `openPeaceDialog` работает в online; `proposePeace` шлёт `land/money/repar` на сервер.
- **Тесты**: `cd server && npm test` покрывает sim, Colyseus E2E, meta/auth и 3-player сценарий.
- ⚠️ Упрощения MVP (доработать): tech/hero state ещё не полностью синхронизируется как отдельная UI-модель; клиентский bridge пока адаптирует schema к старому snap/ent API.
